# Anti-patterns

Things that break the voice on contact. If a draft contains any of these, rewrite before publishing.

## Vocabulary to avoid

- **Hype words:** powerful, revolutionary, game-changing, seamless, unleash, supercharge, amazing, incredible, cutting-edge, world-class, next-generation, robust, leverage, delve, elevate.
- **Corporate filler:** at the end of the day, moving forward, synergy, ecosystem (unless literal), holistic, best-in-class.
- **Fake urgency:** blazingly fast, lightning-fast, in a flash, in no time.
- **Manufactured novelty:** "In today's fast-paced world", "In the ever-evolving landscape of", "Now more than ever".

## Structural anti-patterns

- **TL;DR at the top.** The opening hook already does this. Don't duplicate.
- **"In this post, we will..."** preambles. Skip them. Start with the hook.
- **"In conclusion, we learned that..."** Conclusions reiterate the *why*, not restate the sections.
- **Walls of code.** Break up long snippets with one or two sentences of prose.
- **Bold-in-every-paragraph.** Emphasis loses meaning when used too often.
- **Nested bullet lists three levels deep.** Flatten them.

## Spelling anti-patterns

Use Singapore English (British spelling). Never the American form.

| Avoid (American)   | Use (Singapore/British) |
|--------------------|-------------------------|
| organization       | organisation            |
| behavior           | behaviour               |
| color              | colour                  |
| center             | centre                  |
| analyze            | analyse                 |
| optimize           | optimise                |
| license (noun)     | licence                 |
| defense            | defence                 |
| catalog            | catalogue               |
| favorite           | favourite               |
| traveled           | travelled               |

## Voice anti-patterns

- **Over-qualifying.** "arguably", "potentially", "it could be said that", "one might argue". Cut them. If you believe it, say it.
- **Stacked adverbs.** "really very quickly" → pick one, or better, pick a stronger verb.
- **Passive voice in prose.** "The bug was fixed" → "I fixed the bug".
- **Explaining what's obvious.** "A terminal is a program that..." — trust the reader.
- **Manufactured personal stories.** Only cite your own work when it's the genuine source.

## Formatting anti-patterns

- **Emoji in prose.** No 🚀 😊 💡 ✨ anywhere in the body.
- **Multiple exclamation points.** Prefer a stronger sentence.
- **ALL CAPS for emphasis.** Use inline code or italics.
- **Empty headings.** If a section has one sentence, fold it into the parent.
- **Em-dash overuse.** Em dashes are a seasoning, not a staple. A rough ceiling is roughly one per long paragraph, or one per short section. If two em dashes land in the same paragraph, rewrite one as a full stop or a comma. Never use an em dash just because a sentence felt short.
  - Before: "I use Better Auth — the same library that powers the studio login — because it handles rotation for me."
  - After: "I use Better Auth because it handles rotation for me. It is the same library that powers the studio login."
